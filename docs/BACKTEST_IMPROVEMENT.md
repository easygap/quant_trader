# 백테스트 결과 개선 가이드

최근 스코어링 전략 백테스트(예: `backtest_scoring_20260320_101136.txt`)에서 **총 수익률 -0.25%, 손익비 0.83**이 나온 경우를 기준으로, 원인과 개선 방향을 정리한 문서입니다.

---

## 1. 리포트에서 보이는 문제 요약

| 지표 | 현재 값 | 의미 |
|------|---------|------|
| **손익비** | 0.83 | 1 미만 = 총손실 > 총이익. 승률 53.8%인데도 손해 — **평균 손실(약 5.1만원) > 평균 수익(약 3.6만원)** |
| **상승장 초과수익** | -10.8%p | 코스피가 +10.7% 올랐는데 전략은 -0.1% — **상승장에서 수익을 못 따고 있음** |
| **샤프** | -1.72 | 위험 대비 수익이 매우 나쁨 |
| **한 건의 큰 손실** | STOP_LOSS -8.50% (2025-12-31) | 손실 한 건이 여러 번의 작은 수익을 상쇄 |

정리하면:

1. **손실 거래가 수익 거래보다 커서** 손익비가 1 미만.
2. **상승장에서 전략이 시장을 따르지 못함** — 진입이 늦거나, 익절/트레일링이 너무 빨리 걸려 수익을 잘라먹는 가능성.
3. **큰 손실 1건**(-8.5%)이 전체 성과를 깎아먹음 — 손절이 넓거나, 애초에 그런 진입을 줄일 필요가 있음.

---

## 2. 개선해야 할 부분 (우선순위)

### 2.1 신호 품질 — 가중치·지표 검증 (가장 중요)

현재 `strategies.yaml`의 스코어링 **가중치는 미검증 직관값**입니다. 이 상태로는 같은 구조를 유지해도 손익비가 1 이상으로 바뀐다는 보장이 없습니다.

**권장 파이프라인 (문서/설계서와 동일):**

```bash
# 1) 지표 간 상관 확인 — 고상관 쌍 제거/비활성화
python main.py --mode check_correlation --symbol 005930 --validation-years 5

# 2) 가중치+임계값 그리드 탐색 (고상관 자동 비활성화 포함)
python main.py --mode optimize --strategy scoring --include-weights --auto-correlation --symbol 005930

# 3) OOS 샤프 ≥ 1.0 게이트 통과한 조합만 strategies.yaml에 반영 후, 워크포워드 검증
python main.py --mode validate --walk-forward --strategy scoring --symbol 005930 --validation-years 5
```

- `check_correlation`으로 **다중공선성**을 줄이고,  
- `optimize --include-weights --auto-correlation`으로 **실제 데이터에 맞는 가중치·임계값**을 찾은 뒤,  
- **validate --walk-forward**로 기간별 안정성을 확인하는 것이 핵심입니다.

---

### 2.2 손절·익절 구조 — 손익비 개선

- **손익비 0.83**은 “잃을 때는 크게, 벌 때는 작게” 구조입니다.
- **옵션 A — 손절 강화**: `risk_params.yaml`  
  - `stop_loss.atr_multiplier: 2.0` → `1.5` 등으로 줄여 **한 건 최대 손실**을 줄인다.  
  - 또는 `stop_loss.type: "fixed"`, `fixed_rate: 0.05` 등으로 고정 비율 손절을 시험.
- **옵션 B — 익절/트레일 완화**: 상승장에서 너무 일찍 잘리는지 확인.  
  - `take_profit.partial_target`(현 4%), `trailing_stop.fixed_rate`(현 5%)를 **소폭 상향**해 보유 구간을 늘려 본다. (과매매가 늘지 않도록 과하지 않게.)

실제로는 **2.1에서 가중치를 먼저 정리한 뒤**, 같은 백테스트 기간으로 손익비가 어떻게 바뀌는지 보고, 그다음에 손절/익절만 미세 조정하는 순서를 권합니다.

---

### 2.3 진입 selectivity — 나쁜 진입 줄이기

- **매수 임계값 상향**: `scoring.buy_threshold: 2` → `2.5` 또는 `3`  
  - 진입 횟수는 줄고, “조금만 좋아도 들어가는” 신호가 줄어들어 **손실 거래 비중**을 낮출 수 있습니다.
- **dynamic_threshold**가 켜져 있으면**, 변동성 높을 때 이미 진입이 엄격해지므로**, 가중치 최적화와 함께 그 효과가 백테스트에 어떻게 나오는지 확인하면 좋습니다.

---

### 2.4 상승장 대응

- 상승장에서 **-10.8%p 초과손실**은 “시장은 올랐는데 전략은 못 따랐다”는 뜻입니다.
- 가능한 원인:  
  - 진입이 늦어서 (스코어 임계값이 너무 높거나, 신호가 늦게 나옴)  
  - 부분 익절(4%)·트레일링(5%)이 너무 빨리 걸려 이익을 자른 경우  
  - 또는 **진입 자체가 적어서** 상승 구간을 비어 있는 경우  
- **우선 2.1 가중치/임계값 최적화**로 진입·청산 타이밍이 개선되는지 보고,  
- 그다음 **국면별(상승/횡보/하락) 성과**가 리포트에 어떻게 나오는지 확인한 뒤,  
- 필요하면 `take_profit`/`trailing_stop`을 소폭 완화하는 실험을 하는 흐름이 좋습니다.

---

### 2.5 백테스트 설정 점검

- **backtest_universe.mode**: `historical`(또는 `kospi200`)로 두어 **생존자 편향**을 줄였는지 확인.
- **데이터 구간**: 가능하면 **최소 3~5년**, 상승·하락·횡보가 모두 포함되도록 해서 한 구간에만 맞춘 파라미터인지 검토.
