## 1. 시스템 개요

자동 주식 매매 시스템(알고리즘 트레이딩)은 **사람의 감정 없이** 데이터와 수학적 로직으로 매매 결정을 내리는 프로그램입니다.

### 1.1 핵심 처리 흐름

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 01       │   │ 02       │   │ 03       │   │ 04       │   │ 05       │   │ 06       │
│ 데이터   │──▸│ 지표     │──▸│ 신호     │──▸│ 리스크   │──▸│ 주문     │──▸│ 모니터링 │
│ 수집     │   │ 계산     │   │ 생성     │   │ 관리     │   │ 실행     │   │          │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
  실시간 주가     기술적 지표로    매수/매도       손절/익절      증권사 API로    실시간 성과
  거래량, 뉴스    시장 상태 분석   신호 판단       라인 설정      자동 주문       추적 및 로깅
```

### 1.2 시스템 목표

| 항목 | 목표 |
|------|------|
| **자동화** | 24시간 무인 매매 가능 (장 시간 내 자동 실행) |
| **감정 제거** | 공포/탐욕 없는 규칙 기반 매매 |
| **리스크 제한** | 최대 낙폭(MDD) 15% 이내 제어 |
| **수익 목표** | 연 20% 이상 (백테스팅 기준) |
| **대응 속도** | 실시간 시세 반영 (1초 이내 분석·주문) |

## 3. 핵심 기술 지표

### 3.1 RSI (상대강도지수, Relative Strength Index)

- **설명**: 가격이 얼마나 빠르게 움직이는지 측정. 0~100 사이 값.
- **공식**: `RSI = 100 - (100 / (1 + RS))`  
  - RS = 평균 상승폭 / 평균 하락폭
- **기본 기간**: 14일

| 신호 | 조건 |
|------|------|
| 🟢 매수 | RSI < 30 → 과매도 구간 |
| 🔴 매도 | RSI > 70 → 과매수 구간 |

> **보완**: RSI 단독 사용 시 강한 추세장에서는 오신호가 많음. **다이버전스**(가격은 신저가인데 RSI는 그보다 높은 경우) 확인을 병행하면 신뢰도가 크게 향상됩니다.

---

### 3.2 MACD (이동평균 수렴·확산)

- **설명**: 추세의 방향과 강도를 동시에 파악하는 지표.
- **공식**: 
  - `MACD선 = EMA(12) - EMA(26)`
  - `Signal선 = EMA(MACD, 9)`
  - `히스토그램 = MACD - Signal`
- **기본 기간**: 12/26/9

| 신호 | 조건 |
|------|------|
| 🟢 매수 | MACD선이 Signal선을 상향 돌파 (골든크로스) |
| 🔴 매도 | MACD선이 Signal선을 하향 돌파 (데드크로스) |

> **보완**: 히스토그램의 **기울기 변화**를 함께 관찰하면 신호의 조기 감지가 가능합니다. 또한 **0선 돌파** 여부를 추가 필터로 사용하면 거짓 신호를 줄일 수 있습니다.

---

### 3.3 볼린저 밴드 (Bollinger Bands)

- **설명**: 가격 변동성을 밴드로 표현. 밴드 이탈 시 되돌아오는 특성 활용.
- **공식**: 
  - `중심선 = MA(20)`
  - `상단선 = MA(20) + 2σ (표준편차)`
  - `하단선 = MA(20) - 2σ`
- **기본 기간**: 20일 이동평균

| 신호 | 조건 |
|------|------|
| 🟢 매수 | 가격이 하단 밴드 터치 후 반등 |
| 🔴 매도 | 가격이 상단 밴드 터치 후 하락 |

> **보완**: **밴드폭(Bandwidth)**이 좁아지면 곧 큰 변동이 올 수 있음을 의미합니다 ("스퀴즈" 패턴). 이 패턴을 감지하여 포지션 진입 타이밍을 잡는 전략도 유효합니다.

---

### 3.4 이동평균선 (MA, Moving Average)

- **설명**: 일정 기간 평균 가격. 추세 방향 파악의 기본.
- **공식**: `MA(n) = (P₁ + P₂ + ... + Pₙ) / n`
- **주요 기간**: 5일 / 20일 / 60일 / 120일 / 200일

| 신호 | 조건 |
|------|------|
| 🟢 매수 | 단기선(5일)이 장기선(20일)을 상향 돌파 (골든크로스) |
| 🔴 매도 | 단기선(5일)이 장기선(20일)을 하향 돌파 (데드크로스) |

> **보완**: 단순 이동평균(SMA) 대신 **지수 이동평균(EMA)**을 사용하면 최근 가격에 더 큰 가중치를 부여하여 반응 속도가 빨라집니다. **200일선**은 장기 추세의 기준선으로 널리 사용되므로 추세 필터로 반드시 포함할 것을 권장합니다.

---

### 3.5 거래량 (Volume) & OBV

- **설명**: 가격 움직임을 거래량이 뒷받침하는지 확인.
- **공식**: `OBV = 전일 OBV + (상승 시 +거래량, 하락 시 -거래량)`
- **비교 기준**: 20일 평균 거래량 대비 상대적 비교

| 신호 | 조건 |
|------|------|
| 🟢 매수 | 가격 상승 + 거래량 급증 → 강한 매수 세력 유입 |
| 🔴 매도 | 가격 하락 + 거래량 급증 → 강한 매도 압력 |

> **보완**: **VWAP(거래량 가중 평균가)**도 기관 투자자들이 많이 사용하는 지표입니다. 현재가 > VWAP이면 매수 유리, 현재가 < VWAP이면 매도 유리로 판단합니다.

---

### 3.6 스토캐스틱 (Stochastic Oscillator)

- **설명**: 현재 가격이 최근 가격 범위의 어디에 있는지 측정.
- **공식**: `%K = (현재가 - n일 최저) / (n일 최고 - n일 최저) × 100`
- **기본 기간**: 5/3/3

| 신호 | 조건 |
|------|------|
| 🟢 매수 | %K < 20 이고 %D 상향 돌파 |
| 🔴 매도 | %K > 80 이고 %D 하향 돌파 |

---

### 3.7 ADX (보완 추가 — 평균 방향 지수)

- **설명**: 추세의 **강도**를 측정하는 지표 (방향은 알 수 없음). 0~100 사이 값.
- **공식**: ADX = 14일 DX의 이동평균
- **기본 기간**: 14일

| 조건 | 의미 |
|------|------|
| ADX < 20 | 추세 약함 (횡보장) → 추세 추종 전략 비효율적 |
| ADX > 25 | 추세 강함 → 추세 추종 전략 유효 |
| ADX > 40 | 매우 강한 추세 → 진입 타이밍 주의 (과열 가능) |

> **활용**: 추세 추종 전략의 필수 필터. ADX가 낮을 때는 평균 회귀 전략, 높을 때는 추세 추종 전략으로 전환하는 **적응형 전략**을 구성할 수 있습니다.

---

### 3.8 ATR (보완 추가 — 평균 실질 범위)

- **설명**: 가격의 변동성 크기를 측정. 손절폭 설정에 핵심적으로 사용.
- **공식**: `ATR(n) = n일간 True Range의 이동평균`
  - True Range = max(고가-저가, |고가-전일종가|, |저가-전일종가|)
- **기본 기간**: 14일

> **활용**: 고정 비율 손절(-3%) 대신 `ATR × 배수`로 변동성에 따라 동적 손절폭을 설정하면 급변하는 시장 상황에 더 잘 적응합니다. 예: 손절가 = 매수가 - ATR(14) × 2

## 4. 매매 전략 로직

### 4.1 멀티 지표 스코어링 전략 (초급 ⭐)

> 여러 지표를 점수화하여 합산 점수로 매수/매도를 결정하는 가장 직관적인 전략

**매수 스코어링 규칙:**

| 조건 | 점수 |
|------|------|
| RSI < 30 (과매도) | +2점 |
| MACD 골든크로스 | +2점 |
| 볼린저 하단 이탈 후 반등 | +1점 |
| 거래량이 20일 평균 대비 150% 이상 | +1점 |
| 5일선 > 20일선 (단기 상승 추세) | +1점 |

**매도 스코어링 규칙:** (각 조건 반대 시 동일 점수 감점)

**실행 기준:**
- 총점 ≥ **5점** → 🟢 **매수 실행**
- 총점 ≤ **-4점** → 🔴 **매도 실행**

> **보완**: 점수 임계값(5점, -4점)은 백테스팅을 통해 최적화해야 합니다. 또한 **가중치 방식**(각 지표의 중요도에 따라 점수 차등 부여)도 고려할 수 있습니다.

---

### 4.2 평균 회귀 전략 (중급 ⭐⭐)

> 가격이 평균에서 크게 벗어났을 때 되돌아오는 통계적 특성을 활용

**로직:**

1. 20일 이동평균 및 표준편차 계산
2. `Z-Score = (현재가 - 평균) / 표준편차`
3. Z-Score < **-2** → 과도한 하락 → 🟢 매수
4. Z-Score > **+2** → 과도한 상승 → 🔴 매도
5. RSI로 추가 확인 (이중 필터)
6. 포지션 크기 = Z-Score에 비례하여 조정 (벗어남이 클수록 더 큰 포지션)

> **보완**: 평균 회귀 전략은 **횡보장(ADX < 20)**에서 효과적이지만, 강한 추세장에서는 큰 손실을 초래할 수 있습니다. **반드시 ADX 필터를 병행**해야 합니다. 또한 **공적분(Cointegration) 분석**을 통해 페어 트레이딩으로 확장할 수 있습니다.

---

### 4.3 추세 추종 전략 (중급 ⭐⭐)

> 강한 추세가 형성되면 그 방향으로 따라가는 "추세는 친구다" 전략

**로직:**

1. ADX > 25 → 강한 추세 존재 확인
2. 가격이 200일선 위 → 상승 추세로 판단
3. MACD 히스토그램 양수 전환 → 🟢 매수 진입
4. ATR(14) × 2 → 손절 폭 설정 (변동성 기반)
5. 트레일링 스탑으로 수익 보호
6. 추세 반전 시 즉시 청산

> **보완**: 
> - **브레이크아웃 전략** 추가 — n일간 최고가 돌파 시 매수 (돈키안 채널 활용)
> - **다중 타임프레임 분석** — 주봉에서 추세 확인 후 일봉/시봉에서 진입 타이밍을 찾으면 승률이 향상됩니다
> - **필터링**: 거래량이 평균 이상일 때만 신호를 유효하게 처리

---

### 4.4 볼륨 프로파일 전략 (보완 추가 — 중급 ⭐⭐)

> 특정 가격대에서의 거래량 집중도를 분석하여 지지/저항 수준을 파악

**핵심 개념:**
- **HVN (High Volume Node)**: 거래량이 집중된 가격대 → 강한 지지/저항
- **LVN (Low Volume Node)**: 거래량이 적은 가격대 → 가격이 빠르게 통과

**로직:**
1. 최근 20일간 가격대별 거래량 분포 계산
2. HVN을 지지/저항선으로 설정
3. 가격이 HVN 지지선 접근 + RSI 과매도 → 매수
4. 가격이 HVN 저항선 접근 + RSI 과매수 → 매도

## 5. 리스크 관리

> [!CAUTION]
> 리스크 관리는 전략보다 더 중요합니다. 아무리 좋은 전략이라도 리스크 관리 없이는 반드시 파산합니다.

### 5.1 포지션 사이징 — 1% 룰

- **규칙**: 한 종목에서 최대 손실은 전체 자본의 **1%**를 초과하지 않음
- **공식**: `주문수량 = (자본 × 0.01) / (매수가 - 손절가)`
- **예시**: 자본 1,000만원, 매수가 50,000원, 손절가 48,500원  
  → 수량 = (10,000,000 × 0.01) / (50,000 - 48,500) = **66주**

---

### 5.2 손절매 (Stop Loss) ✂️

- **규칙**: 매수가 대비 **-3~5%** 하락 시 자동 손절. 큰 손실 방지
- **고정 손절**: `손절가 = 매수가 × (1 - 0.03)`
- **변동성 기반 손절 (권장)**: `손절가 = 매수가 - ATR(14) × 2`

> **보완**: 고정 비율 손절보다 **ATR 기반 동적 손절**이 시장 상황에 더 잘 적응합니다. 변동성이 큰 종목은 손절폭을 넓히고, 안정적인 종목은 좁히는 방식입니다.

---

### 5.3 익절매 (Take Profit) 🎯

- **규칙**: 매수가 대비 **+6~15%** 상승 시 자동 익절
- **공식**: `익절가 = 매수가 × (1 + 0.10)`
- **손익비 원칙**: 손절 3% 대비 익절은 최소 6% 이상 (손익비 ≥ 2:1)

> **보완**: 고정 익절보다는 **부분 익절** 전략이 효과적입니다 — 목표가 1차 도달 시 50% 매도, 나머지 50%는 트레일링 스탑으로 추가 수익을 추구합니다.

---

### 5.4 트레일링 스탑 (Trailing Stop) 📈

- **규칙**: 수익 구간에서 고점 대비 **-3%** 하락 시 매도. 수익 보호
- **공식**: `매도가 = max(고가 기록) × (1 - 0.03)`
- **동작**: 가격이 올라가면 스탑가도 함께 올라감. 내려가면 스탑가 유지

> **보완**: ATR 기반 트레일링 스탑(`고점 - ATR × 배수`)이 더 시장 적응적입니다. 또한 파라볼릭 SAR(Parabolic SAR) 지표를 트레일링 스탑으로 활용하는 방법도 있습니다.

---

### 5.5 분산 투자 🌐

- **규칙**: 단일 종목 최대 투자 비중 **20%**, 섹터별 집중 방지
- **공식**: `종목당 투자금 = 총자본 × 0.20 이하`

> **보완 — 상세 분산 규칙:**
>
> | 제한 항목 | 기준 |
> |-----------|------|
> | 단일 종목 | 최대 20% |
> | 단일 섹터 | 최대 40% |
> | 최대 동시 보유 | 5~10 종목 |
> | 현금 비중 | 최소 20% 유지 (비상 매수 대비) |

---

### 5.6 최대 낙폭 (MDD) 제한 🚨

- **규칙**: 전체 포트폴리오 **-15%** 도달 시 모든 매매 일시 **중단**
- **공식**: `MDD = (고점 - 저점) / 고점 × 100`
- **복귀 조건**: MDD 회복 후 50% 규모로 축소 재개

> **보완**: MDD 제한은 반드시 **시스템 레벨에서 강제**되어야 합니다 (전략 로직이 이를 무시하지 못하도록). 또한 **일일 최대 손실 한도**(-3%)도 설정하여 하루에 과도한 손실을 방지해야 합니다.

---

### 5.7 블랙스완 대응 (보완 추가)

| 이벤트 유형 | 대응 방안 |
|------------|-----------|
| 시장 급락 (-5% 이상) | 모든 포지션 즉시 청산 (서킷브레이커 발동 전 탈출) |
| API 장애 | 수동 주문 가능한 백업 수단 마련 |
| 네트워크 장애 | 로컬 감시 프로세스로 비상 시 사전 등록된 손절 주문 실행 |
| 서버 다운 | 증권사 서버측 예약 주문(조건부 주문)을 병행 설정 |

## 7. 백테스팅 & 검증

### 7.1 핵심 성과 지표 (KPI)

| 지표 | 설명 | 목표치 |
|------|------|--------|
| **총 수익률** | 전략의 전체 기간 누적 수익 | > 20% / 년 |
| **샤프 지수 (Sharpe Ratio)** | 위험 대비 수익 효율성 (높을수록 좋음) | > 1.5 |
| **최대 낙폭 (MDD)** | 최고점 대비 최대 하락폭 | < -20% |
| **승률 (Win Rate)** | 수익 거래 / 전체 거래 비율 | > 50% |
| **손익비 (Profit Factor)** | 평균 수익 / 평균 손실 | > 2.0 |
| **거래 횟수** | 백테스팅 기간 내 총 거래수 | 통계적 유의성 확보 (최소 100회) |
| **칼마 비율 (Calmar Ratio)** | 연 수익률 / MDD (보완 추가) | > 1.0 |

### 7.2 검증 절차

```
단계 1  과거 3~5년치 데이터 다운로드
  │
단계 2  훈련 세트(70%)와 검증 세트(30%) 분리
  │
단계 3  훈련 데이터로 전략 파라미터 최적화
  │
단계 4  검증 데이터로 과적합 여부 확인
  │
단계 5  워크포워드 분석으로 견고성 검증
  │       (6개월 훈련 → 1개월 검증을 슬라이딩 윈도우로 반복)
  │
단계 6  거래비용 반영
  │       ├── 매매 수수료: 0.015%
  │       └── 매도 세금: 0.2% (코스피 / 코스닥)
  │

  │

  │

```

## 9. 예외 처리 및 안정성 강화 로직 (v2 핵심 추가)

실전 환경에서 증권사 API 제한과 네트워크 불안정을 극복하기 위한 고도화된 방어 로직입니다.

### 9.1 API 호출 최적화 및 보호
- **API Rate Limiter (Token Bucket)**: 증권사 API 호출 한도(초당 N건)를 제어하여 계정 Ban 방지.
- **HTTP/네트워크 에러 재시도**: 429(Too Many Requests), 50x 에러 발생 시 지수 백오프(Exponential Backoff) 및 `Retry-After` 헤더를 파싱하여 자동 재요청.
- **Circuit Breaker 패턴**: API 서버의 장기간 장애 시 연속 실패를 감지하고, 상태를 OPEN시켜 추가적인 API 호출을 즉각 차단 및 알림 발송.

### 9.2 웹소켓 무중단 수신
- **Auto-Reconnect 루프**: 네트워크 순단으로 수신이 끊길 시 지정된 간격으로 즉각 무한 재연결 시도 수행.
- **Heartbeat (좀비 커넥션 방지)**: 데이터 수신이 일정 시간(예: 45초) 없으면 자체적으로 Ping을 검사하고 응답이 없으면 연결을 강제로 재생성.

### 9.3 데이터 및 알림 무결성
- **수신 데이터 정합성 검증 (Validator)**: API 응답 및 실시간 웹소켓 가격에 섞인 Null/NaN, 음수 주가 등을 사전 필터링하여 오작동 방지.
- **알림 이중화(디스코드 + 이메일)**: 핵심 알림 메신저(디스코드 등) 장애 시나 API 차단 같은 치명적 오류 파악 시, 등록된 이메일 계정으로 Fallback 알림 발송.
- **시크릿 환경변수 격리**: API 키 등 민감값과 주요 파라미터를 하드코딩하지 않고 `.env` 파일과 `os.environ`을 통해 동적 주입.

## 10. 향후 개선 사항

- **정적 자산 배분(EMP) 리밸런싱 시스템**: 타사 퀀트(Fintics) 방식처럼, "성장주 40%, 배당주 40%, 채권 20%" 식의 고정 목표 비중(Target Weight)을 자산 맵에 설정하고, 주기적으로 초과/미달 비중을 맞추도록 붓거나 나누어 담는 자동 리밸런싱 기능 구현.
- **다중 오실레이터 기반 과매수/과매도 필터 개선**: 기존 이동평균 기반 모멘텀 외에 RSI, CCI, Stochastic, Williams %R 4종 보조지표를 혼합 적용하여 "과매수 시 매수 금지, 과매도 구간일 때만 분할 매수"하는 정교한 투표(Voting) 스코어링 지표 도입.
- **딥러닝 / ML 예측 모델 연동**: 현재 룰베이스/스코어링 기반에서 LSTM 등 AI 기반 예측 모델로 고도화
- **웹 대시보드 (GUI)**: 실시간 계좌 상태, 수익률 차트, 포트폴리오 관리를 현재의 데몬 스크립트와 분리하여 FastAPI + React/Streamlit 웹 브라우저에서 모니터링 (Hot-Reloading 전략파라미터 지원 포함)
- **멀티 증권사 지원**: KIS 외 영웅문(Kiwoom) 등 타 증권사 API 연동 아키텍처 확장

## 9. 주의사항

### 🚨 치명적 주의 (Critical)

> [!CAUTION]
> **과적합 위험**  
> 과거 데이터에 너무 최적화하면 실전에서 실패합니다. 반드시 미래 데이터(Out-of-Sample)로 검증할 것. 파라미터 수가 많을수록 과적합 위험이 급증합니다. 단순한 전략이 종종 복잡한 전략보다 실전에서 더 잘 작동합니다.

> [!CAUTION]
> **블랙스완 이벤트**  
> 코로나, 전쟁, 금융위기 같은 극단적 사건은 어떤 알고리즘도 예측 불가합니다. **비상 손절 매커니즘**과 **현금 비중 유지**가 생존의 핵심입니다.

### ⚠️ 경고 (Warning)

> [!WARNING]
> **소액으로 시작**  
> 반드시 **페이퍼 트레이딩(가상매매) 3개월** → **소액 실전 테스트 3개월** → **점진적 증액** 순서로 진행하세요. 처음부터 전 재산을 투입하는 것은 도박과 다를 바 없습니다.

> [!WARNING]
> **수수료 함정**  
> 잦은 매매(고빈도 거래)는 수수료와 세금으로 수익이 사라집니다. 편도 수수료 0.015% + 매도 세금 0.2% = 왕복 약 0.23%이므로, 매매 1회당 최소 0.5% 이상의 기대 수익이 있어야 손익분기를 넘깁니다.

### ℹ️ 참고 (Info)

> [!NOTE]
> **법적 규제**  
> 한국 금융법상 자동매매는 **개인 계좌에서만** 허용됩니다. 타인의 자금을 대리 운용하는 것은 **투자자문업 미등록** 으로 불법이며, 처벌 대상입니다.

> [!NOTE]
> **세금 신고**  
> 국내 주식 양도소득세(대주주 기준), 해외 주식 양도소득세(연 250만원 초과분 22%)를 반드시 확인하고, 자동매매 수익에 대한 세금 신고를 빠짐없이 해야 합니다.

> [!NOTE]
> **시스템 안정성**  
> 자동매매 시스템은 **장 시간 동안 중단 없이** 동작해야 합니다. 클라우드 서버(AWS, GCP) 또는 고정 IP가 있는 항시 가동 환경(NAS, 미니PC 등)에서 운영하는 것을 권장합니다. 가정용 PC에서 운영 시 예기치 못한 재부팅, 네트워크 장애 등의 위험이 있습니다.

## 부록: 용어 정리 (보완 추가)

| 용어 | 설명 |
|------|------|
| **EMA** | 지수 이동평균. 최근 데이터에 더 높은 가중치를 부여 |
| **SMA** | 단순 이동평균. 기간 내 모든 데이터에 동일 가중치 |
| **골든크로스** | 단기선이 장기선을 상향 돌파 (매수 신호) |
| **데드크로스** | 단기선이 장기선을 하향 돌파 (매도 신호) |
| **슬리피지** | 주문 시점의 예상 가격과 실제 체결 가격의 차이 |
| **서킷브레이커** | 시장 급변 시 거래를 일시 정지하는 제도 |
| **MDD** | Maximum Drawdown. 고점 대비 최대 하락폭 |
| **샤프 지수** | (수익률 - 무위험수익률) / 수익률의 표준편차. 위험 대비 효율 |
| **워크포워드** | 과거 데이터를 슬라이딩 윈도우로 분할하여 반복 검증하는 방법 |
| **페이퍼 트레이딩** | 실제 돈 없이 가상으로 시뮬레이션 매매하는 것 |
| **Z-Score** | 데이터가 평균에서 표준편차 몇 배만큼 떨어졌는지를 나타내는 수치 |
| **OBV** | On Balance Volume. 거래량과 가격 방향을 결합한 지표 |
| **ATR** | Average True Range. 변동성 크기를 측정하는 지표 |
| **ADX** | Average Directional Index. 추세 강도를 측정하는 지표 |
| **VWAP** | Volume Weighted Average Price. 거래량 가중 평균 가격 |
