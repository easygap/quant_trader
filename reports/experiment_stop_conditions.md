## 즉시 중단 (Hard Stop)

### H1. 코드/설정 변조 감지
- Git HEAD가 `c182823` 이 아닌 경우
- Config Hash가 `1366a00b19c4aa58` 이 아닌 경우
- **조치**: 변조 원인 파악 → 원복 후 재개 가능 (중단 기간만큼 실험 연장)

### H2. 실매매 환경 노출
- `ENABLE_LIVE_TRADING=true` 가 .env에 설정된 경우
- KIS API가 mock이 아닌 실전 URL로 연결된 경우
- **조치**: 즉시 서비스 중단, .env 확인 및 수정, 실매매 실행 여부 확인

### H3. 데이터 무결성 붕괴
- DB 파일 손상 (`sqlite3 data/quant_trader.db "PRAGMA integrity_check"` → `ok` 아닌 경우)
- 5영업일 연속 데이터 수집 실패 (FDR + yfinance 모두)
- **조치**: DB 백업 복원 또는 데이터 소스 복구 후 재개

### H4. 시스템 장기 다운
- 연속 3영업일 이상 서비스 미실행 (로그 기준)
- **조치**: 원인 파악 후 재개, 누락 기간만큼 실험 연장

## 경고 후 판단 (Soft Stop)

### S1. 포트폴리오 MDD 15% 도달
- `risk_params.yaml`의 `max_portfolio_mdd: 0.15` 발동
- 시스템이 자동으로 신규 매수를 중단함
- **판단**:
  - MDD 15% 도달만으로 실험 중단하지 않음 (시스템 방어 동작 관측 목적)
  - MDD 20% 도달 시 실험 중단 검토
  - MDD 25% 도달 시 **즉시 중단**

### S2. 일일 손실 3% 3회 이상 발생
- 실험 기간 중 일일 손실 3% 초과가 3회 이상
- **판단**: 원인 분석 (시장 급락 vs 전략 문제) → 시장 급락이면 계속, 전략 버그면 중단

### S3. 전략 성과 열화 감지
- `performance_degradation` 발동: 최근 20거래 승률 35% 미만
- 시스템이 자동으로 신규 매수를 중단함
- **판단**: 관측 계속 (자동 방어 동작 검증 목적), 20영업일 연속 발동 시 중단 검토

### S4. 블랙스완 3회 이상 발동
- 실험 기간 중 블랙스완 감지가 3회 이상 발동
- **판단**: 각 발동 시점의 시장 상황 기록, 시스템 대응 정상 여부 확인

### S5. Paper vs Backtest 괴리
- `paper_backtest_compare` 수익률 차이 15%p 초과, 또는 승률 차이 20%p 초과
- **판단**: 구현 버그 의심 → 원인 분석 (데이터 차이 vs 코드 문제)

| 상황 | 이유 |
|------|------|

| 블랙스완 1~2회 발동 | 시스템 방어 동작 검증 |
